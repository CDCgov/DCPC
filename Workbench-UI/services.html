Services

This information is geared towards the expert user

The clinical pipelines developed on the CLEW workbench are available as services. All use open-source NLP tools from the CLEW Instance of the LAPPS grid. They are customized to solve a particular use case. These services can be run in real-time by an external application calling the service to process a block of clinical text from a pathology report. Other clinical documents will need services tailored for the processing objectives of the user.

All available services in the list below include the following information:

  • Discussion of the use case the service was built to solve including a discussion of identified semantics
  • The approach of the service usage to address the use case
  • Documentation so you can make adjustments as needed based on your use case
  • A demo with discussion

Performance

For maximum performance and accuracy of results, semantic definitions must match the semantics of how they are identified for a particular service. For example, the lexicons for the Cancer Pathology and Safety Surveillance services below apply for each to their particular use cases. However, while there would be a sacrifice in the accuracy of results, these pipelines should be applied cautiously to other use cases.
